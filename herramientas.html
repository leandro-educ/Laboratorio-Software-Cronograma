Lenguaje y runtime

Java 21 como base del laboratorio.

Maven para compilar, testear y empaquetar.

Arquitectura real

Spring Boot, Docker y RabbitMQ en la segunda mitad del curso.

01

Stack recomendado

Un stack coherente evita fricción innecesaria y deja la energía del grupo en lo importante: diseñar y construir buen software.

Editor

Visual Studio Code o IntelliJ IDEA

  • Autocompletado y navegación por clases.
  • Integración con Git.
  • Terminal incorporada para flujo continuo.

Calidad

JUnit + JaCoCo

  • Tests unitarios y de regresión.
  • Cobertura como evidencia complementaria.
  • Trabajo incremental desde el inicio.

Integración

Docker + Postman + RabbitMQ

  • Contenedores para servicios auxiliares.
  • Prueba manual de endpoints.
  • Mensajería distribuida y trazabilidad.
02

Configuración sugerida del entorno

Una secuencia simple para preparar la notebook antes de las primeras prácticas.

Paso 1

Instalar Java 21

Verificar que el sistema reconozca correctamente el JDK.

java -version
javac -version

Paso 2

Instalar Maven

Será la base para compilar, ejecutar tests y organizar dependencias.

mvn -version

Paso 3

Configurar Git

El laboratorio parte del trabajo versionado. Cada evidencia debería quedar registrada en repositorio.

git --version
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

Paso 4

Preparar herramientas complementarias

DB Browser o SQLite, Docker Desktop, Postman y un navegador para RabbitMQ Management.

docker --version
03

Estructura base de un proyecto Maven

Una estructura mínima, clara y suficiente para comenzar con orden desde la primera semana.

Vista general

laboratorio-software/
├─ pom.xml
├─ README.md
├─ src/
│  ├─ main/
│  │  └─ java/
│  └─ test/
│     └─ java/
└─ .gitignore

¿Qué aporta cada parte?

pom.xmlDefine dependencias, plugins y ciclo de build.
src/main/javaContiene el código fuente productivo.
src/test/javaAloja los tests automatizados.
README.mdDocumenta ejecución, decisiones y pasos de reproducción.
04

Comandos útiles durante la cursada

Un pequeño repertorio operativo para no perder tiempo cuando la clase entra en ritmo de laboratorio.

Maven 
mvn clean test
mvn clean package
mvn spring-boot:run
Docker + RabbitMQ 
docker compose up -d
docker ps
docker compose down

Consejo de cursada

Documentar cómo se ejecuta el proyecto y cómo se verifican los tests ahorra tiempo al docente, al equipo y al propio estudiante cuando vuelve a abrir su trabajo semanas después.

Error frecuente

Esperar a la semana del parcial para ordenar el repositorio. En laboratorio, la organización técnica también forma parte del aprendizaje.
